---
description: Runs the typical git command cycle for file changes in the current local repo
---

Perform the necessary `git ...` commands to: 

1. Add/track/stage all file changes.
2. Commit tracked/staged changes (with a resonable, auto-generated commit message).
3. Push the commit to the remote repository.
4. Check the `git status` of the local repository
5. Inform the user of the results (e.g., all succeeded, some failed, etc.)
